# GreenSpark: Análisis financiero

**Equipo:** HackHeroes · **Mención:** Energía · **Lugar:** Santa Cruz de la Sierra, Bolivia

> **Criterio de honestidad:** GreenSpark todavía no cuenta con cotizaciones cerradas, entrevistas suficientes ni mediciones locales para afirmar precios definitivos, ahorro energético o rentabilidad observada. Este documento diferencia las hipótesis financieras de los datos que deben validarse antes de presentar una oferta comercial.

## 0. Cómo leer los números

- **[SUPUESTO]:** valor provisional utilizado para probar la lógica del modelo financiero.
- **[A COTIZAR]:** costo que requiere una propuesta comercial o consulta a proveedores.
- **[A VALIDAR]:** hipótesis que exige entrevista, medición o piloto.
- **[EJEMPLO HIPOTÉTICO]:** cálculo ilustrativo; no representa un resultado observado ni una promesa al cliente.

Todos los montos se expresan en **bolivianos (`Bs`)** y deben actualizarse con evidencia local.

## 1. Resumen ejecutivo

GreenSpark propone un modelo **B2B por fases**. La primera venta no depende de producir electricidad comercial: una universidad privada puede pagar por investigación aplicada, simulación, métricas de sostenibilidad y una hoja de ruta experimental. Después, el proyecto puede monetizar pilotos MFC instrumentados, monitoreo y reportes. En la fase 3, si el volumen segregado y la demanda lo justifican, puede evaluar biodigestores o soluciones híbridas.

| Elemento | Definición inicial |
| --- | --- |
| **Modelo de negocio** | Servicio B2B institucional con ingreso recurrente y proyectos de implementación por fase. |
| **Cliente inicial** | Universidad privada de Santa Cruz de la Sierra. |
| **Usuario** | Equipo universitario de investigación, sostenibilidad o innovación. |
| **Pagador** | Institución que financia el convenio o licencia. |
| **Ingreso principal de fase 1** | Convenio mensual de investigación aplicada, simulación y reporte. |
| **Costo principal de fase 1** | Mantenimiento técnico, acompañamiento institucional y soporte. |
| **Punto de equilibrio operativo de referencia** | **3 instituciones activas** bajo los supuestos del escenario base. |

## 2. Alcance comercial y mercado inicial

GreenSpark no presenta a toda la ciudad como mercado inmediato. La entrada comercial se concentra en instituciones capaces de financiar aprendizaje aplicado antes de invertir en infraestructura.

| Alcance | Segmento | Cómo se valida |
| --- | --- | --- |
| **TAM** | Instituciones y empresas con residuos bioorgánicos y necesidad de evaluar valorización energética en Bolivia. | Investigación sectorial y segmentación por tipo de generador. |
| **SAM** | Universidades privadas, colegios privados, restaurantes y agroindustrias de Santa Cruz de la Sierra con interés y capacidad operativa para participar. | Censo comercial, entrevistas y registro de volumen segregable. |
| **SOM inicial** | Universidades privadas dispuestas a financiar una primera fase investigativa. | Al menos tres entrevistas y una carta de interés o convenio exploratorio. |

No se asigna un número de instituciones al TAM, SAM o SOM hasta verificar fuentes locales. Para el escenario financiero se usa una curva de adquisición explícita y reemplazable.

## 6. Unit economics y punto de equilibrio

| Métrica | Fórmula | Resultado del escenario base |
| --- | --- | ---: |
| **ARPU mensual** | ingreso promedio por institución | `Bs 3.000` |
| **Costo variable unitario** | costo por institución activa | `Bs 900` |
| **Margen bruto unitario** | `3.000 - 900` | `Bs 2.100` |
| **Margen bruto porcentual** | `2.100 / 3.000` | `70%` |
| **Costos fijos mensuales** | suma de costos fijos | `Bs 6.300` |
| **Break-even operativo** | `6.300 / 2.100` | **`3 instituciones activas`** |
| **CAC de referencia** | esfuerzo comercial por institución adquirida | `Bs 3.500` **[SUPUESTO]** |
| **Retención utilizada para planificar** | horizonte conservador, no churn medido | `6 meses` **[SUPUESTO]** |
| **LTV de contribución** | `2.100 × 6` | `Bs 12.600` |
| **Relación LTV/CAC** | `12.600 / 3.500` | `3,6x` |

El break-even es operativo: no incluye todavía el costo del piloto físico ni infraestructura de fase 3. La relación LTV/CAC debe recalcularse cuando existan ventas, renovaciones y costos comerciales observados.

## 7. ROI y payback para el cliente

### 7.1 Fase 1: valor institucional

El retorno de una universidad no debe confundirse con ahorro energético garantizado. La primera validación consiste en determinar si la institución paga por investigación aplicada, participación estudiantil, métricas de sostenibilidad y reducción de incertidumbre antes de invertir.

### 7.2 Fases 2 y 3: valor energético medido

La tarifa aplicable debe tomarse de la factura vigente y categoría real del cliente. CRE publica estructuras diferenciadas por categoría; no corresponde utilizar una tarifa única para todo proyecto.

```text
Ahorro energético validado =
  kWh medidos de autoconsumo
  × tarifa aplicable
  - costos de operación y mantenimiento
```

```text
Beneficio neto mensual del cliente =
  ahorro energético validado
  + valor operativo cuantificable
  - costo mensual de GreenSpark
```

```text
Payback =
  inversión inicial del cliente
  / beneficio neto mensual validado
```

| Inversión inicial | Beneficio neto mensual | Payback resultante |
| ---: | ---: | ---: |
| `Bs 6.000` | `Bs 1.000` | `6 meses` |
| `Bs 6.000` | `Bs 2.000` | `3 meses` |
| `Bs 6.000` | `Bs 3.000` | `2 meses` |

> **[EJEMPLO HIPOTÉTICO]:** la tabla muestra sensibilidad matemática. No representa una cotización, un ahorro medido ni una promesa comercial.

## 11. Riesgos financieros y mitigaciones

| Riesgo | Impacto | Mitigación |
| --- | --- | --- |
| El precio institucional supera la disposición a pagar. | Alto | Entrevistar universidades y ajustar el alcance antes de fijar tarifa. |
| Los costos del piloto MFC son mayores a lo esperado. | Alto | Cotizar materiales, sensores y caracterización antes de vender fase 2. |
| La adquisición de clientes es más lenta que la curva base. | Alto | Ejecutar un piloto comercial acotado y revisar CAC después de cada entrevista. |
| El soporte consume más horas por institución. | Medio-Alto | Definir entregables mensuales, automatizar reportes y medir carga operativa. |
| La API de IA externa aumenta costos. | Medio | Mantener el LLM como componente opcional y medir costo por reporte. |
| El aporte energético no justifica inversión física. | Alto | Separar el valor investigativo de fase 1 y escalar únicamente después de medir. |

## 15. Conclusión

GreenSpark presenta una hipótesis económica defendible, no una rentabilidad inventada. El modelo inicial puede alcanzar equilibrio operativo con tres instituciones bajo supuestos transparentes. La siguiente decisión no es escalar infraestructura: es validar disposición a pagar, costo de adquisición y cotización del piloto. Solo después corresponde calcular ahorro energético, ROI y payback con mediciones locales, la factura real del cliente y la revisión regulatoria aplicable.
